## 4. ネットワーク & 配信

### Amazon VPC (Virtual Private Cloud)

**概要**: AWS内に構築する仮想ネットワーク。

#### VPC の基本コンポーネント

| コンポーネント | 説明 | 試験のポイント |
|----------------|------|---------------|
| **サブネット** | VPC内のIPアドレス範囲 | パブリック/プライベートに分割 |
| **ルートテーブル** | トラフィックの経路制御 | サブネットに関連付け |
| **インターネットゲートウェイ (IGW)** | VPCとインターネットの接続点 | パブリックサブネットに必須 |
| **NAT Gateway** | プライベートサブネットからのインターネットアクセス | Outboundのみ許可 |
| **VPCエンドポイント** | インターネット経由せずAWSサービスへ接続 | セキュリティ向上 |

#### VPC エンドポイント

| タイプ | 対象 | 特徴 | 試験のポイント |
|--------|------|------|---------------|
| **ゲートウェイ型** | S3、DynamoDB | ルートテーブルにエントリ追加 | 無料 |
| **インターフェース型 (PrivateLink)** | その他のサービス | ENIを作成 | 有料、プライベート接続 |

> 🎯 **試験のポイント**: 「インターネット経由せずにS3にアクセス」→ **VPCエンドポイント（ゲートウェイ型）**

#### VPC 間接続

| 方法 | 特徴 | 試験のポイント |
|------|------|---------------|
| **VPCピアリング** | 1対1接続、推移的ルーティング不可 | シンプルな接続 |
| **Transit Gateway** | ハブ型接続、多数のVPCを一元管理 | フルメッシュ問題の解決 |
| **AWS PrivateLink** | サービスをプライベートに公開 | SaaSへのプライベート接続 |

> ⚠️ **重要**: VPCピアリングは**推移的ルーティング不可**（A-B、B-CがつながってもA-Cは直接つながらない）

#### オンプレミス接続

| 方法 | 特徴 | 導入時間 | 試験のポイント |
|------|------|----------|---------------|
| **Site-to-Site VPN** | インターネット経由、IPsec暗号化 | 数分〜 | すぐに接続したい、低コスト |
| **Direct Connect** | 専用線、安定した帯域 | 数週間〜 | 大容量、高セキュリティ |
| **Direct Connect + VPN** | 専用線上にVPN | - | 高セキュリティ + 暗号化 |

#### VPC セキュリティ

| 機能 | レベル | ステート | 試験のポイント |
|------|--------|----------|---------------|
| **セキュリティグループ** | インスタンス | ステートフル | 戻り通信は自動許可 |
| **ネットワークACL** | サブネット | ステートレス | 戻り通信も明示的に許可必要 |

#### VPC トラブルシューティング・監視

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **VPC Flow Logs** | トラフィック情報をキャプチャ | 「REJECT通信の調査」 |
| **Traffic Mirroring** | パケットをコピーして分析 | 「ペイロードまで検査」 |
| **Reachability Analyzer** | 設定から接続可否を分析 | 「通信できない原因特定」 |

#### その他のネットワーク機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **Egress-Only IGW** | IPv6専用、Outboundのみ | IPv6でのNAT的な役割 |
| **DHCP Option Sets** | DNSサーバー等を設定 | オンプレDNSを使用 |

### Amazon Route 53

**概要**: 高可用性のDNSサービス。ドメイン登録、ルーティング、ヘルスチェック機能。

#### ルーティングポリシー

| ポリシー | 説明 | 試験のポイント |
|----------|------|---------------|
| **シンプル** | 標準的なルーティング | 単一リソース |
| **加重 (Weighted)** | 重み付けで分散 | A/Bテスト、段階的移行 |
| **レイテンシー** | 最も遅延の少ないリージョンへ | グローバルユーザー |
| **フェイルオーバー** | アクティブ/パッシブ構成 | DR、Sorry Page |
| **位置情報 (Geolocation)** | ユーザーの国/地域で分岐 | コンテンツのローカライズ |
| **地理的近接性 (Geoproximity)** | 距離＋バイアス値で分岐 | トラフィックシフト |
| **複数値回答** | 複数IPをランダムに返す | 簡易な負荷分散 |
| **IPベース** | クライアントIPで分岐 | 特定企業IPへの対応 |

> 🎯 **試験のポイント**:
>
> - 「メインサイトダウン時にS3のSorry Pageへ」→ **フェイルオーバー**
> - 「ユーザーに最も近いリージョンへ」→ **レイテンシー** または **地理的近接性**

#### レコードタイプ

| タイプ | 説明 | 試験のポイント |
|--------|------|---------------|
| **A** | ドメイン → IPv4 | 基本 |
| **AAAA** | ドメイン → IPv6 | IPv6対応 |
| **CNAME** | ドメイン → 別ドメイン | Zone Apexに使用不可 |
| **Alias** | ドメイン → AWSリソース | Zone Apexに使用可能、無料 |

> ⚠️ **重要**: ルートドメイン（Zone Apex: example.com）にはCNAMEは使えない → **Aliasレコード**を使用

#### Route 53 の高度な機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **ヘルスチェック** | エンドポイントの監視 | フェイルオーバーの前提 |
| **Traffic Flow** | 複雑なルーティングを視覚的に管理 | 複数ポリシーの組み合わせ |
| **Resolver** | VPCとオンプレ間のDNS連携 | ハイブリッド環境 |
| **DNS Firewall** | DNSクエリのフィルタリング | 悪意あるドメインをブロック |
| **Application Recovery Controller (ARC)** | フェイルオーバーの準備状況監視 | 高度なDR制御 |

### Amazon CloudFront

**概要**: グローバルCDN（コンテンツ配信ネットワーク）。エッジロケーションでコンテンツをキャッシュ。

#### CloudFront の重要機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **OAC / OAI** | S3へのアクセスをCloudFront経由のみに制限 | S3直接アクセス防止 |
| **署名付きURL/Cookie** | コンテンツへの一時的アクセス | 有料会員向け配信 |
| **地理的制限** | 国単位でアクセス制御 | コンテンツのライセンス |
| **Invalidation（無効化）** | キャッシュの強制削除 | 即時コンテンツ更新 |
| **Origin Groups** | フェイルオーバー構成 | オリジンの高可用性 |
| **Field-Level Encryption** | 特定フィールドを暗号化 | 機密データの保護 |

#### エッジでのコード実行

| 機能 | 言語 | 実行時間 | 用途 |
|------|------|----------|------|
| **CloudFront Functions** | JavaScript | 1ms未満 | ヘッダー操作、URLリライト |
| **Lambda@Edge** | Node.js, Python | 最大5-30秒 | 複雑な処理、外部DB参照 |

> 🎯 **試験のポイント**:
>
> - 「S3へのアクセスをCloudFront経由のみに」→ **OAC + バケットポリシー**
> - 「軽量なヘッダー操作」→ **CloudFront Functions**
> - 「外部APIを呼び出す複雑な処理」→ **Lambda@Edge**

### Elastic Load Balancing (ELB)

**概要**: トラフィックを複数のターゲットに分散するロードバランサー。

#### ロードバランサーの種類

| タイプ | レイヤー | 特徴 | 試験のポイント |
|--------|---------|------|---------------|
| **ALB** | L7（HTTP/HTTPS） | パスベース、ホストベースルーティング | Webアプリケーション |
| **NLB** | L4（TCP/UDP） | 超低レイテンシー、固定IP | 高性能、静的IP必要 |
| **GLB** | L3（ゲートウェイ） | サードパーティアプライアンス連携 | ファイアウォール製品 |
| **CLB** | L4/L7 | レガシー | 新規利用非推奨 |

#### ELB の重要機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **ヘルスチェック** | ターゲットの正常性確認 | 異常時は自動除外 |
| **スティッキーセッション** | 同じターゲットに継続接続 | ステートフルアプリ |
| **コネクションドレイニング** | 終了前に処理完了を待つ | 安全なスケールイン |
| **クロスゾーン負荷分散** | AZをまたいで均等に分散 | ALBはデフォルト有効 |

> 🎯 **試験のポイント**:
>
> - 「固定IP」「超低レイテンシー」→ **NLB**
> - 「パスベースルーティング」「WAF連携」→ **ALB**
> - 「サードパーティファイアウォール」→ **GLB**

### AWS Global Accelerator

**概要**: AWSグローバルネットワーク経由でアプリへのアクセスを最適化。2つの固定IPを提供。

| 比較項目 | CloudFront | Global Accelerator |
|----------|------------|-------------------|
| 用途 | 静的コンテンツのキャッシュ | 動的コンテンツの高速化 |
| 固定IP | なし | あり |
| キャッシュ | あり | なし |

> 🎯 **試験のポイント**: 「固定IPが必要」+「動的コンテンツ」+「グローバル高速化」→ **Global Accelerator**

---

### AWS Transit Gateway

**概要**: 多数のVPCやオンプレミスネットワークを中央ハブとして接続。

> 🎯 **試験のポイント**:
>
> - 「VPCピアリングがフルメッシュで複雑」→ **Transit Gateway**
> - 「推移的なルーティングが必要」→ **Transit Gateway**

---

### AWS Network Firewall

**概要**: VPC向けのマネージドファイアウォール。ステートフルパケット検査、IPS機能。

> 🎯 **試験のポイント**: 「VPCの出入り口で詳細なパケットフィルタリング」→ **Network Firewall**
