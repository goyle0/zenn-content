## はじめに

Markdownを使っていると「こういうことできないかな？」と思う場面がありますよね。

この記事では、意外と知られていない**Markdownの便利な小技**をまとめました。PDF変換時の改ページから、折りたたみ、脚注まで、知っておくと役立つTipsを紹介します。

## 1. 改ページ（PDF変換時）

MarkdownをPDFに変換するとき、**任意の位置で改ページ**したい場合があります。

### 書き方

改ページしたい位置の**直前**に以下を挿入します。

```html
<div style="page-break-before:always"></div>
```

### 使用例

```markdown
## 第1章 はじめに

ここに第1章の内容...

<div style="page-break-before:always"></div>

## 第2章 本題

ここから新しいページで始まる
```

:::message
この記法はPDF変換ツール（Pandoc、VSCode拡張など）で有効です。Zennのプレビューでは効果がありません。
:::

## 2. 折りたたみ（アコーディオン）

長いコードや補足情報を**折りたたんで表示**できます。

### 書き方

```html
<details>
<summary>クリックして展開</summary>

ここに折りたたむ内容を書く

</details>
```

### 表示例

<details>
<summary>クリックして展開</summary>

これは折りたたまれた内容です。
クリックすると表示されます。

```python
# コードも入れられます
print("Hello, World!")
```

</details>

### ポイント

- `<summary>` と `</summary>` の間がクリック可能なタイトル
- 内容部分の前後に**空行が必要**
- Markdown記法も内部で使用可能

## 3. 脚注・注釈

本文中に**注釈**を入れて、ページ下部に説明を表示できます。

### 書き方

```markdown
本文中に脚注を入れます[^1]。複数も可能です[^note]。

[^1]: ここに脚注の内容を書きます。
[^note]: 名前付きの脚注も使えます。
```

### 表示例

Markdownは軽量マークアップ言語です[^1]。John Gruberによって開発されました[^2]。

[^1]: HTMLに変換可能なテキスト記法のこと。
[^2]: 2004年に公開された。

### ポイント

- `[^名前]` で本文中にマーク
- `[^名前]: 説明` で注釈内容を定義（ファイル内のどこでもOK）
- 表示順は本文での登場順に自動で番号が振られる

## 4. コメント（非表示テキスト）

公開されたページには**表示されない**メモを残せます。

### 書き方

```markdown
<!-- これはコメントです。表示されません -->

本文はここから始まります。

<!-- 
複数行のコメントも
書けます
-->
```

### 使いどころ

- 自分用のメモ
- 下書き段階の文章
- 共同編集時の申し送り事項

## 5. 改行のコントロール

Markdownでの改行は環境によって挙動が異なります。

### 方法1: 行末にスペース2つ

```markdown
1行目（末尾にスペース2つ）  
2行目
```

### 方法2: HTMLタグ

```html
1行目<br>2行目
```

### 方法3: 空行（段落分け）

```markdown
1つ目の段落

2つ目の段落（間に空行）
```

:::message
Zennでは通常の改行がそのまま反映されますが、GitHubなど他のプラットフォームでは挙動が異なる場合があります。
:::

## 6. チェックボックス（タスクリスト）

タスク管理に便利な**チェックボックス**を作成できます。

### 書き方

```markdown
- [ ] 未完了のタスク
- [x] 完了したタスク
- [ ] もう1つのタスク
```

### 表示例

- [ ] 記事の下書き
- [x] 画像の準備
- [ ] 公開前の最終チェック

### ポイント

- `[ ]` で未チェック
- `[x]` でチェック済み
- `-` と `[` の間にスペースが必要

## 7. 取り消し線

**訂正**や**変更履歴**を示すときに使えます。

### 書き方

```markdown
~~取り消したいテキスト~~
```

### 表示例

価格: ~~5,000円~~ → 3,980円（20%OFF）

## 8. 画像サイズの指定（Zenn限定）

Zennでは画像の**幅をピクセル単位**で指定できます。

### 書き方

```markdown
![代替テキスト](画像URL =300x)
```

### 例

```markdown
![ロゴ](/images/logo.png =200x)
```

### ポイント

- `=数値x` で幅を指定（高さは自動調整）
- URLと `=` の間に**半角スペース**が必要

## 9. Diffコードブロック

コードの**変更箇所**をわかりやすく表示できます。

### 書き方

````markdown
```diff
- 削除された行
+ 追加された行
  変更なしの行
```
````

### 表示例

```diff
- const message = "Hello";
+ const message = "Hello, World!";
  console.log(message);
```

### 言語指定と組み合わせ（Qiita等）

一部のプラットフォームでは、言語指定と組み合わせられます。

````markdown
```diff javascript
- console.log("old");
+ console.log("new");
```
````

## 10. 水平線（区切り線）

セクションの区切りに**水平線**を引けます。

### 書き方（どれでもOK）

```markdown
---

***

___
```

### 表示例

上のセクション

---

下のセクション

## 11. エスケープ（特殊文字をそのまま表示）

Markdown記号を**記号として表示**したい場合。

### 書き方

```markdown
\# これは見出しにならない
\*アスタリスクをそのまま表示\*
\`バッククォートをそのまま表示\`
```

### 表示例

\# これは見出しにならない

\*アスタリスクをそのまま表示\*

## まとめ

| Tips | 用途 | よく使う場面 |
|------|------|-------------|
| 改ページ | PDF変換時のページ区切り | レポート、ドキュメント |
| 折りたたみ | 長い内容を隠す | ログ、補足情報 |
| 脚注 | 注釈・参考情報 | 技術記事、論文風 |
| コメント | 非公開メモ | 下書き、共同編集 |
| チェックボックス | タスク管理 | TODO、進捗管理 |
| Diff | 変更箇所の表示 | コードレビュー |

これらのTipsを活用して、より読みやすいドキュメントを作成しましょう！
