## 1. コンピューティング & サーバーレス

### Amazon EC2 (Elastic Compute Cloud)

**概要**: AWS上の仮想サーバー。OS、インスタンスタイプ（CPU、メモリ最適化など）を選択して利用。

#### インスタンスタイプの覚え方

| ファミリー | 用途 | 覚え方 |
|-----------|------|--------|
| **M** (汎用) | バランスの取れたワークロード | **M**edium（中間） |
| **C** (コンピューティング最適化) | 高性能Webサーバー、バッチ処理 | **C**ompute |
| **R** (メモリ最適化) | インメモリDB、リアルタイム分析 | **R**AM |
| **T** (バースト可能) | 開発環境、小規模Web | **T**urbo burst |
| **I/D** (ストレージ最適化) | DWH、分散ファイルシステム | **I**O / **D**isk |
| **P/G** (GPU) | 機械学習、グラフィックス | **P**arallel / **G**raphics |

#### 購入オプション

| オプション | 特徴 | 割引率 | 試験のポイント |
|-----------|------|--------|---------------|
| **オンデマンド** | 従量課金、いつでも起動/停止可能 | なし | 予測できないワークロード |
| **リザーブドインスタンス (RI)** | 1年/3年契約で割引 | 最大72% | 24時間365日稼働する本番システム |
| **Savings Plans** | 使用量をコミット（柔軟性あり） | 最大72% | RIより柔軟、ファミリー変更可能 |
| **スポットインスタンス** | 余剰容量を低価格で | 最大90% | 中断可能なバッチ処理、CI/CD |
| **Dedicated Hosts** | 物理サーバー専有 | - | BYOLライセンス、コンプライアンス |
| **Dedicated Instances** | 専用ハードウェア（制御なし） | - | 物理共有禁止だがホスト管理不要 |

> 🎯 **試験のポイント**:
>
> - 「コスト削減」+「24時間稼働」→ **RI または Savings Plans**
> - 「コスト削減」+「中断可能」→ **スポットインスタンス**
> - 「ソケット単位のライセンス」→ **Dedicated Hosts**

#### プレイスメントグループ

| タイプ | 配置 | 用途 | 試験のポイント |
|--------|------|------|---------------|
| **クラスター** | 単一AZ・単一ラック | HPC、低レイテンシー | 「ノード間通信を最速に」 |
| **スプレッド** | 異なるラック（最大7台/AZ） | 少数の重要インスタンス | 「同時障害を絶対に防ぐ」 |
| **パーティション** | 論理的なパーティションに分散 | Hadoop、Cassandra、Kafka | 「大規模分散システム」 |

> ⚠️ **注意**: 稼働中のインスタンスを後からプレイスメントグループに追加することは**できない**。AMIを作成して再作成が必要。

#### その他の重要機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **ユーザーデータ** | 起動時に自動実行されるスクリプト | ブートストラップ処理 |
| **インスタンスメタデータ** | `169.254.169.254`で自身の情報取得 | スクリプト内でIP取得等 |
| **ハイバネーション（休止）** | メモリをEBSに保存して停止 | 「起動時間を短縮したい」 |
| **Instance Store** | 物理直結の一時ストレージ | 「消えても良いキャッシュ」専用 |

### Auto Scaling (Amazon EC2 Auto Scaling)

**概要**: 需要に応じてEC2の台数を自動で増減（水平スケーリング）し、可用性とコスト効率を維持。

#### スケーリングポリシー

| ポリシー | 説明 | 試験のポイント |
|----------|------|---------------|
| **ターゲット追跡** | 特定メトリクス（CPU50%等）を維持 | 最も推奨される方式 |
| **ステップ** | 閾値ごとに増減数を変える | 段階的なスケーリング |
| **シンプル** | 1つの閾値で増減 | レガシー、非推奨 |
| **スケジュール** | 時間指定で増減 | 予測可能なピーク対応 |
| **予測** | 機械学習で需要予測 | トラフィックパターンが安定 |

#### 重要な設定項目

| 項目 | 説明 | 試験のポイント |
|------|------|---------------|
| **クールダウン** | スケーリング後の待機時間 | 頻繁なスケーリング防止 |
| **ライフサイクルフック** | 起動/終了時に一時停止してカスタム処理 | 「終了前にログをS3に退避」 |
| **ウォームプール** | 事前に起動済みインスタンスをプール | 起動時間短縮 |

> 🎯 **試験のポイント**:
>
> - 「SQSのメッセージ数に応じてスケール」→ **Backlog Per Instance（カスタムメトリクス）**
> - 「インスタンス終了前にログ保存」→ **ライフサイクルフック**

### AWS Lambda

**概要**: サーバー管理不要でコードを実行するサーバーレスコンピューティング。イベント駆動型。

#### 基本仕様

| 項目 | 値 | 試験のポイント |
|------|-----|---------------|
| 最大実行時間 | **15分（900秒）** | 15分超え → Batch/ECS |
| メモリ | 128MB〜10GB | メモリ増でCPUも増加 |
| 一時ストレージ | /tmp に最大10GB | 一時ファイル処理用 |
| 同時実行数 | デフォルト1,000/リージョン | スロットリング注意 |

#### 権限の理解（頻出！）

| 権限タイプ | 方向 | 説明 | 例 |
|-----------|------|------|-----|
| **実行ロール** | Lambda → 外部 | Lambdaが他サービスを操作 | DynamoDBへ書き込み |
| **リソースベースポリシー** | 外部 → Lambda | 他サービスがLambdaを呼び出す | S3イベントでInvoke |

> ⚠️ **頻出トラブル**: 「S3イベントでLambdaが起動しない」→ **リソースベースポリシー**の許可漏れ

#### Lambda のデプロイ・バージョン管理

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **Layers** | 共通ライブラリを複数関数で共有 | 依存関係の管理 |
| **Versions** | 不変のスナップショット | 本番用バージョンを固定 |
| **Aliases** | バージョンへのポインタ（DEV, PROD等） | カナリアリリース（トラフィック分割） |

### Amazon ECS / EKS / Fargate

**概要**: コンテナオーケストレーションサービス群。

| サービス | 説明 | 試験のポイント |
|----------|------|---------------|
| **ECS** | Dockerコンテナ管理（AWS独自） | シンプル、AWS統合が容易 |
| **EKS** | Kubernetes管理サービス | K8s経験者、マルチクラウド |
| **Fargate** | サーバーレスコンテナ実行基盤 | 「OSパッチ不要」がキーワード |

> 🎯 **試験のポイント**:
>
> - 「コンテナ使いたいがサーバー管理したくない」→ **Fargate**
> - 「既存のKubernetesをAWSで」→ **EKS**

---

### AWS Elastic Beanstalk

**概要**: アプリケーションコードをアップロードするだけで、EC2、LB、Auto Scaling等を自動構築するPaaS。

| 構成要素 | 説明 |
|----------|------|
| **Application** | 論理的なトップレベルのコンテナ |
| **Environment** | 実行環境（Web Server / Worker） |
| **Version** | デプロイするアプリのバージョン |

> 🎯 **試験のポイント**: 「インフラ管理を最小限にしてコード開発に集中したい」→ **Elastic Beanstalk**

---

### AWS Batch

**概要**: フルマネージド型のバッチ処理実行サービス。ジョブ量に応じてEC2/Fargateを自動プロビジョニング。

> 🎯 **試験のポイント**:
>
> - 「Lambdaの15分制限を超えるバッチ処理」→ **AWS Batch**
> - 「数千件のバッチジョブを効率的に」→ **AWS Batch**

---

### Amazon Lightsail

**概要**: 月額固定料金で簡単に使えるVPS（仮想プライベートサーバー）。

> 🎯 **試験のポイント**: 「シンプルで安価なWebサーバー」「複雑な構成は不要」→ **Lightsail**
