## 2. ストレージ

### Amazon S3 (Simple Storage Service)

**概要**: 容量無制限のオブジェクトストレージ。耐久性99.999999999%（イレブンナイン）。

#### ストレージクラス比較

| クラス | 用途 | 最小保存期間 | 取り出し料金 | 試験のポイント |
|--------|------|-------------|-------------|---------------|
| **S3 Standard** | 高頻度アクセス | なし | なし | デフォルト |
| **S3 Intelligent-Tiering** | アクセスパターン不明 | なし | なし | 自動で階層移動 |
| **S3 Standard-IA** | 低頻度アクセス | 30日 | あり | 月1回程度のアクセス |
| **S3 One Zone-IA** | 低頻度・単一AZ | 30日 | あり | 再作成可能なデータ |
| **S3 Glacier Instant Retrieval** | アーカイブ（即時取得） | 90日 | あり | 四半期に1回アクセス |
| **S3 Glacier Flexible Retrieval** | アーカイブ（数時間） | 90日 | あり | 年1-2回アクセス |
| **S3 Glacier Deep Archive** | 長期アーカイブ | 180日 | あり | 最安、12時間以上 |

#### Glacier 取り出しオプション（頻出！）

| オプション | 時間 | コスト | 試験のポイント |
|-----------|------|--------|---------------|
| **迅速 (Expedited)** | 1-5分 | 高い | 緊急時の即時取得 |
| **標準 (Standard)** | 3-5時間 | 中間 | デフォルト |
| **大容量 (Bulk)** | 5-12時間 | 最安 | 大量データの低コスト取得 |

#### S3 の重要機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **バージョニング** | オブジェクトの世代管理 | 誤削除・上書きからの復旧 |
| **MFA Delete** | 削除時にMFA必須 | ルートユーザーのみ設定可能 |
| **Object Lock** | WORM（書込み1回、読取り複数回） | コンプライアンス要件 |
| **ライフサイクルポリシー** | 自動でストレージクラス移行/削除 | コスト最適化 |
| **レプリケーション (CRR/SRR)** | 別リージョン/同一リージョンに複製 | DR対策、コンプライアンス |
| **Transfer Acceleration** | エッジロケーション経由で高速化 | 遠隔地からのアップロード |

#### S3 Object Lock モード

| モード | 説明 | 試験のポイント |
|--------|------|---------------|
| **ガバナンスモード** | 特別な権限で解除可能 | テスト環境など |
| **コンプライアンスモード** | 誰も解除不可（ルートユーザーも） | 法的要件で「絶対削除禁止」 |

#### S3 セキュリティ

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **バケットポリシー** | バケット単位のアクセス制御 | クロスアカウントアクセス |
| **ACL** | オブジェクト単位の制御（レガシー） | 無効化が推奨 |
| **Object Ownership** | オブジェクト所有者の統一 | 「他アカウントのファイルを削除できない」問題の解決 |
| **ブロックパブリックアクセス** | 公開設定を一括ブロック | セキュリティのベストプラクティス |
| **署名付きURL** | 一時的なアクセス許可 | 有料会員への期間限定配信 |
| **署名付きCookie** | 複数ファイルへの一時アクセス | ストリーミング配信 |

#### S3 パフォーマンス最適化

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **マルチパートアップロード** | 大きなファイルを分割アップロード | 100MB以上で推奨 |
| **Byte-Range Fetches** | 並列ダウンロード | 大容量ダウンロードの高速化 |
| **S3 Select** | SQLでフィルタリングして取得 | 転送量削減 |

#### S3 管理機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **S3 Inventory** | オブジェクト一覧をCSV出力 | 監査、分析用 |
| **S3 Batch Operations** | 大量オブジェクトに一括処理 | タグ付け、コピー、Lambda実行 |
| **S3 Storage Lens** | 組織全体のS3使用状況を可視化 | マルチアカウントの管理 |

> 🎯 **試験のポイント**:
>
> - 「アクセスパターンが予測不能」→ **Intelligent-Tiering**
> - 「法的要件で削除禁止」→ **Object Lock（コンプライアンスモード）**
> - 「他アカウントがアップしたファイルを削除できない」→ **Object Ownership（バケット所有者強制）**

### Amazon EBS (Elastic Block Store)

**概要**: EC2にアタッチするブロックストレージ。OSやデータベースの保存に使用。

#### ボリュームタイプ

| タイプ | 用途 | IOPS | 試験のポイント |
|--------|------|------|---------------|
| **gp3** | 汎用SSD（推奨） | 最大16,000 | コスト効率が良い |
| **gp2** | 汎用SSD（旧型） | 容量に比例（最大16,000） | バースト性能 |
| **io2 Block Express** | 高性能SSD | 最大256,000 | ミッションクリティカルDB |
| **io1/io2** | プロビジョンドIOPS | 最大64,000 | 高IOPS要件 |
| **st1** | スループット最適化HDD | - | ビッグデータ、DWH |
| **sc1** | コールドHDD | - | 低頻度アクセス、最安 |

#### EBS 重要機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **スナップショット** | S3に増分バックアップ | リージョン間コピー可能 |
| **Snapshot Archive** | 90日以上保存するスナップショット | コスト75%削減 |
| **Recycle Bin** | 削除したスナップショットを復元可能 | 誤削除対策 |
| **暗号化** | KMSで暗号化 | 新規ボリュームはデフォルト暗号化推奨 |
| **Multi-Attach** | 複数EC2に同時アタッチ | io1/io2のみ、クラスター用 |

#### RAID構成

| 構成 | 目的 | 試験のポイント |
|------|------|---------------|
| **RAID 0** | パフォーマンス向上 | 「単一EBSでIOPS不足」 |
| **RAID 1** | 冗長性向上 | EBS自体の障害対策 |

> 🎯 **試験のポイント**:
>
> - 「最高のIOPS性能」→ **io2 Block Express**
> - 「誤って削除したスナップショットを復元」→ **Recycle Bin**

### Amazon EFS (Elastic File System)

**概要**: 複数のEC2から同時マウント可能な共有ファイルストレージ（NFSプロトコル）。**Linux専用**。

#### パフォーマンスモード

| モード | 説明 | 試験のポイント |
|--------|------|---------------|
| **汎用（推奨）** | バランスの取れた性能 | ほとんどのワークロード |
| **最大I/O** | 高スループット、高レイテンシー | 数千クライアント同時アクセス |

#### スループットモード

| モード | 説明 | 試験のポイント |
|--------|------|---------------|
| **バースト** | 容量に応じた基本性能＋バースト | デフォルト |
| **プロビジョンド** | 指定したスループットを確保 | 「容量小だが高性能必要」 |
| **エラスティック** | 需要に応じて自動スケール | 予測不能なワークロード |

#### ストレージクラス

| クラス | 説明 | 試験のポイント |
|--------|------|---------------|
| **Standard** | 高頻度アクセス | デフォルト |
| **Standard-IA** | 低頻度アクセス | コスト削減 |
| **One Zone / One Zone-IA** | 単一AZ | さらにコスト削減 |

> 🎯 **試験のポイント**:
>
> - 「複数EC2でファイル共有（Linux）」→ **EFS**
> - 「容量は小さいが高スループット必要」→ **プロビジョンドスループット**

### Amazon FSx ファミリー

| サービス | ファイルシステム | プロトコル | 試験のポイント |
|----------|----------------|-----------|---------------|
| **FSx for Windows** | Windows File Server | SMB | AD連携、Windows環境 |
| **FSx for Lustre** | Lustre | POSIX | HPC、S3連携 |
| **FSx for NetApp ONTAP** | ONTAP | NFS/SMB/iSCSI | 重複排除、NetApp移行 |
| **FSx for OpenZFS** | OpenZFS | NFS | ZFS移行、スナップショット |

> 🎯 **試験のポイント**:
>
> - 「Windows + AD + SMB」→ **FSx for Windows**
> - 「HPC + S3連携」→ **FSx for Lustre**
> - 「重複排除でコスト削減」→ **FSx for NetApp ONTAP**

### AWS Storage Gateway

**概要**: オンプレミスとAWSストレージを接続するハイブリッドストレージサービス。

| タイプ | プロトコル | 用途 | 試験のポイント |
|--------|-----------|------|---------------|
| **S3 File Gateway** | NFS/SMB | オンプレからS3へファイル保存 | ファイルサーバー移行 |
| **Volume Gateway (Cached)** | iSCSI | 全データAWS、頻繁なデータのみローカルキャッシュ | ストレージ容量節約 |
| **Volume Gateway (Stored)** | iSCSI | 全データローカル、AWSにバックアップ | ローカルアクセス重視 |
| **Tape Gateway** | iSCSI (VTL) | 物理テープの代替 | 既存バックアップソフト維持 |

> 🎯 **試験のポイント**:
>
> - 「オンプレからNFS/SMBでS3にアクセス」→ **S3 File Gateway**
> - 「物理テープを廃止したい」→ **Tape Gateway**

### AWS Snow ファミリー

**概要**: 物理デバイスでの大量データ転送・エッジコンピューティング。

| デバイス | 容量 | 特徴 | 試験のポイント |
|----------|------|------|---------------|
| **Snowcone** | 8TB/14TB | 最小（お弁当箱サイズ） | ドローン、バックパック |
| **Snowball Edge Storage** | 80TB〜 | データ転送特化 | ペタバイト規模の移行 |
| **Snowball Edge Compute** | - | エッジでEC2/Lambda実行 | オフライン処理 |
| **Snowmobile** | 100PB | トラック輸送 | エクサバイト規模 |

> 🎯 **試験のポイント**:
>
> - 「ネットワークが遅く、大量データ移行」→ **Snowball Edge**
> - 「10PB以上」→ **Snowmobile**
