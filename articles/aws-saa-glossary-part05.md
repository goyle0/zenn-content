## 5. セキュリティ & ID管理

### AWS IAM (Identity and Access Management)

**概要**: AWSリソースへのアクセス権限を管理するサービス。

#### IAM の基本要素

| 要素 | 説明 | 試験のポイント |
|------|------|---------------|
| **ユーザー** | 個人（人間）のアカウント | 長期認証情報 |
| **グループ** | ユーザーの集合 | 権限の一括管理 |
| **ロール** | サービスや外部IDに権限を付与 | 一時的な認証情報 |
| **ポリシー** | 権限を定義するJSON文書 | 最小権限の原則 |

#### IAM ロールの使用場面

| 場面 | 説明 | 試験のポイント |
|------|------|---------------|
| **EC2にロールを付与** | インスタンスプロファイル経由 | アクセスキー埋め込み禁止 |
| **クロスアカウントアクセス** | 他アカウントへの権限委任 | AssumeRole |
| **フェデレーション** | 外部IDプロバイダーとの連携 | SAML、OIDC |

> ⚠️ **アンチパターン**: コード内にアクセスキーを埋め込む → **IAMロールを使用**

#### IAM ポリシーの種類

| タイプ | 説明 | 試験のポイント |
|--------|------|---------------|
| **AWS管理ポリシー** | AWSが作成・管理 | 一般的なユースケース |
| **カスタマー管理ポリシー** | ユーザーが作成・管理 | 組織固有の要件 |
| **インラインポリシー** | 特定のプリンシパルに直接埋め込み | 1対1の関係、非推奨 |

#### IAM の高度な機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **IAM Policy Simulator** | ポリシーの動作をテスト | 権限設定の確認 |
| **IAM Access Analyzer** | 外部からアクセス可能なリソースを検出 | セキュリティ監査 |
| **Permission Boundaries** | 権限の上限を設定 | 委任時の権限制限 |

### AWS Organizations

**概要**: 複数のAWSアカウントを一元管理。請求の統合やポリシー（SCP）の適用。

#### SCP (Service Control Policy)

**概要**: 組織内のアカウントに対して、利用可能なサービスやアクションを制限。

| 特徴 | 説明 |
|------|------|
| ホワイトリスト/ブラックリスト | 許可または拒否を指定 |
| 継承 | OUの階層で継承される |
| IAMとの関係 | SCPとIAMの両方で許可が必要 |

> ⚠️ **重要**: SCPで拒否されると、IAMで管理者権限があってもアクセス不可

---

### AWS Control Tower

**概要**: マルチアカウント環境（ランディングゾーン）のセットアップと統制を自動化。

#### ガードレール

| タイプ | 方法 | 説明 |
|--------|------|------|
| **予防的 (Preventive)** | SCP | 操作自体を禁止 |
| **発見的 (Detective)** | AWS Config | 違反を検知・アラート |

> 🎯 **試験のポイント**:
>
> - 「操作をブロックしたい」→ **予防的ガードレール（SCP）**
> - 「違反を検知したい」→ **発見的ガードレール（Config）**

### AWS IAM Identity Center (旧 AWS SSO)

**概要**: 複数のAWSアカウントやアプリケーションへのシングルサインオン（SSO）を一元管理。

> 🎯 **試験のポイント**: 「複数AWSアカウントへのログインを1つのIDで管理」→ **IAM Identity Center**

### AWS Directory Service

**概要**: Active Directory関連のマネージドサービス。

| サービス | 説明 | 試験のポイント |
|----------|------|---------------|
| **Managed Microsoft AD** | 完全なAD機能、信頼関係構築可能 | 最も高機能、オンプレADと連携 |
| **AD Connector** | オンプレADへのプロキシ | クラウドにデータを置かない |
| **Simple AD** | Samba 4ベース、基本機能のみ | 低コスト、信頼関係不可 |

> 🎯 **試験のポイント**:
>
> - 「オンプレADと信頼関係」→ **Managed Microsoft AD**
> - 「クラウドにユーザー情報を置きたくない」→ **AD Connector**

### Amazon Cognito

**概要**: Web/モバイルアプリ向けのユーザー認証サービス。

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **ユーザープール** | ユーザー登録・認証（ID/Pass管理） | 「ログイン機能」 |
| **IDプール** | AWSリソースへのアクセス権限付与 | 「S3にファイルアップロード」 |

> 🎯 **試験のポイント**:
>
> - 「アプリのログイン機能」→ **ユーザープール**
> - 「認証後にS3やDynamoDBにアクセス」→ **IDプール**

### 暗号化サービス

#### AWS KMS (Key Management Service)

**概要**: 暗号化キー（CMK）の作成・管理。データの暗号化に使用。

| キータイプ | 説明 | 試験のポイント |
|-----------|------|---------------|
| **AWS管理キー** | AWSが管理（自動ローテーション） | 手軽、制御は限定的 |
| **カスタマー管理キー** | ユーザーが管理 | 柔軟な制御 |
| **カスタマー提供キー** | ユーザーがキーを持ち込み | 完全な制御 |

#### AWS CloudHSM

**概要**: 専用のハードウェアセキュリティモジュール。

> 🎯 **試験のポイント**: 「専用HSM」「FIPS 140-2 Level 3」→ **CloudHSM**

#### AWS Secrets Manager

**概要**: データベースの認証情報（パスワードなど）を安全に保存し、**自動ローテーション**。

> 🎯 **試験のポイント**: 「DBパスワードの自動ローテーション」→ **Secrets Manager**

#### Systems Manager Parameter Store

**概要**: 設定データやパスワードを階層化して保存。

> 🎯 **比較**:
>
> - 自動ローテーション必要 → **Secrets Manager**
> - シンプルな設定保存、低コスト → **Parameter Store**

#### AWS ACM (Certificate Manager)

**概要**: SSL/TLS証明書の発行・管理・更新を自動化。

> 🎯 **試験のポイント**: 「HTTPS化」「SSL証明書の管理」→ **ACM**

### セキュリティ検出・保護サービス

| サービス | 機能 | 試験のポイント |
|----------|------|---------------|
| **GuardDuty** | 脅威検出（ログ分析） | 「不正アクセス検知」「マルウェア通信」 |

> 🎯 **試験のポイント**:
>
> - 「脅威の検知」→ **GuardDuty**
> - 「検知後の調査」→ **Detective**
> - 「S3内の機密データ」→ **Macie**

### AWS WAF & Shield

#### AWS WAF

**概要**: Webアプリケーションファイアウォール。SQLインジェクション、XSS等から保護。

| 適用先 | 説明 |
|--------|------|
| CloudFront | グローバル配信の保護 |
| ALB | アプリケーションの保護 |
| API Gateway | APIの保護 |

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **地理的一致ルール** | 国単位でブロック/許可 | 「特定の国からのアクセス遮断」 |
| **レートベースルール** | リクエスト数で制限 | DDoS対策の一部 |
| **マネージドルール** | AWSやサードパーティ提供 | 即座に保護を有効化 |

#### AWS Shield

| プラン | 説明 | 試験のポイント |
|--------|------|---------------|
| **Standard** | 無料、自動有効 | 基本的なDDoS保護 |
| **Advanced** | 月額$3,000〜、DRT支援 | 高度なDDoS保護、コスト保護 |

> 🎯 **試験のポイント**:
>
> - 「SQLインジェクション対策」→ **WAF**
> - 「DDoS対策」→ **Shield**
> - 「特定の国をブロック」→ **WAFの地理的一致ルール**
