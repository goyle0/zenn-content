## 6. アプリケーション統合（疎結合）

### Amazon SQS (Simple Queue Service)

**概要**: メッセージキューイングサービス。システム間を「疎結合」にし、非同期処理を実現。

#### キュータイプ

| タイプ | 特徴 | 試験のポイント |
|--------|------|---------------|
| **標準キュー** | 順序保証なし、重複あり、高スループット | 大量メッセージ処理 |
| **FIFOキュー** | 順序保証あり、重複なし、300msg/秒 | 順序が重要な処理 |

#### SQS の重要設定

| 設定 | 説明 | 試験のポイント |
|------|------|---------------|
| **可視性タイムアウト** | 処理中に他から見えなくなる期間 | 「重複処理」→ タイムアウト延長 |
| **ロングポーリング** | メッセージがなくても最大20秒待機 | 空レスポンス削減、コスト削減 |
| **遅延キュー** | 配信を指定時間遅らせる | DB更新完了を待つ |
| **デッドレターキュー (DLQ)** | 処理失敗メッセージの退避先 | 無限ループ防止、調査用 |

> 🎯 **試験のポイント**:
>
> - 「空レスポンスが多くコストがかかる」→ **ロングポーリング**
> - 「同じメッセージが重複処理される」→ **可視性タイムアウト延長**

### Amazon SNS (Simple Notification Service)

**概要**: プッシュ型のメッセージングサービス（Pub/Sub）。複数のサブスクライバーに同時配信。

| サブスクライバー | 用途 |
|-----------------|------|
| SQS | 非同期処理 |
| Lambda | サーバーレス処理 |
| HTTP/HTTPS | Webhook |
| Email | 通知 |
| SMS | モバイル通知 |

#### SNS + SQS ファンアウト構成

**概要**: SNSトピックに送信 → 複数のSQSキューに並列配信。

> 🎯 **試験のポイント**: 「1つのイベントで複数の異なる処理を同時実行」→ **SNS + SQS ファンアウト**

#### SNS FIFO Topics

**概要**: 順序保証と重複排除を持つSNSトピック。SQS FIFOと組み合わせて使用。

### Amazon MQ

**概要**: Apache ActiveMQ / RabbitMQ 向けのマネージド型メッセージブローカー。

> 🎯 **試験のポイント**: 「既存のActiveMQ/RabbitMQをコード変更なしでAWSへ移行」→ **Amazon MQ**

### AWS Step Functions

**概要**: 複数のAWSサービス（Lambda等）を組み合わせたワークフローを視覚的に構築・実行。

> 🎯 **試験のポイント**: 「複数のLambdaを順番に実行するワークフロー」→ **Step Functions**

---

### Amazon EventBridge

**概要**: サーバーレスのイベントバス。SaaSアプリやAWSサービス間のイベントルーティング。

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **ルール** | イベントのフィルタリングとターゲット指定 | イベント駆動アーキテクチャ |
| **Pipes** | フィルタリング・変換のパイプライン | Lambdaなしでデータ変換 |
| **Scheduler** | 大量タスクのスケジュール実行 | 定期実行、Cron |

---

### Amazon API Gateway

**概要**: APIの作成・公開・管理を行うフルマネージドサービス。

| タイプ | 特徴 | 試験のポイント |
|--------|------|---------------|
| **REST API** | フル機能、柔軟性高い | 従来のAPI |
| **HTTP API** | シンプル、低コスト | 軽量なAPI |
| **WebSocket API** | 双方向通信 | チャット、リアルタイム |

#### API Gateway の認証

| 方法 | 説明 | 試験のポイント |
|------|------|---------------|
| **Cognito Authorizer** | Cognitoユーザープールと連携 | ユーザー認証 |
| **Lambda Authorizer** | カスタム認証ロジック | 独自のトークン検証 |
| **IAM** | IAM署名による認証 | AWSサービス間 |

---

### AWS AppSync

**概要**: GraphQL APIをフルマネージドで提供。リアルタイムデータ同期、オフライン機能。

> 🎯 **試験のポイント**: 「GraphQL」「オフライン同期（モバイルアプリ）」→ **AppSync**

## 7. 管理 & 運用

### Amazon CloudWatch

**概要**: AWSリソースの監視サービス。メトリクス、ログ、アラーム。

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **メトリクス** | CPU使用率等の監視 | 基本監視（5分）/ 詳細監視（1分） |
| **Logs** | ログの収集・管理・分析 | ログの一元管理 |
| **Alarms** | 閾値超過時の通知/アクション | Auto Scaling連携 |
| **Events / EventBridge** | イベント駆動の処理 | スケジュール実行 |
| **Dashboards** | カスタムダッシュボード | 可視化 |
| **Dashboard Sharing** | IAMなしでダッシュボード共有 | 社外への共有 |

#### CloudWatch の高度な機能

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **Synthetics (Canaries)** | エンドポイントの外形監視 | リンク切れ、応答時間監視 |
| **RUM** | 実際のユーザー体験を収集 | ブラウザ側のパフォーマンス |
| **Evidently** | 機能フラグ、A/Bテスト | 新機能の段階的ロールアウト |
| **ServiceLens** | X-Rayと統合したサービスマップ | 分散トレーシング |

---

### AWS CloudTrail

**概要**: AWSアカウント内のAPI操作（誰がいつ何をしたか）を記録する監査サービス。

| 特徴 | 説明 |
|------|------|
| デフォルト有効 | 90日間保持 |
| 長期保存 | S3へ出力 |
| 整合性検証 | ログ改ざん検知 |

> 🎯 **試験のポイント**: 「誰がS3バケットを削除したか調査」→ **CloudTrail**

---

### AWS Config

**概要**: AWSリソースの設定変更履歴を記録・評価する構成管理サービス。

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **構成履歴** | リソースの変更履歴 | 「いつ設定が変わったか」 |
| **ルール** | 設定がポリシーに準拠しているか評価 | 「暗号化されていないEBS」検出 |
| **修復アクション** | 違反時に自動修復 | SSM Automationと連携 |
| **アグリゲーター** | 複数アカウント・リージョンを集約 | 組織全体の可視化 |

> 🎯 **試験のポイント**: 「設定変更の履歴」「コンプライアンス違反の検出」→ **AWS Config**

---

### AWS Systems Manager (SSM)

**概要**: EC2や他のリソースの管理・運用を自動化するツール群。

| 機能 | 説明 | 試験のポイント |
|------|------|---------------|
| **Session Manager** | SSH不要でブラウザからEC2接続 | 「踏み台サーバー廃止」「ポート22不要」 |
| **Run Command** | 複数サーバーにコマンド一括実行 | パッチ適用、設定変更 |
| **Patch Manager** | OSパッチの自動適用 | メンテナンスウィンドウ指定 |
| **Parameter Store** | 設定データ・パスワードの保存 | ハードコード禁止 |
| **Automation** | 運用タスクの自動化 | 定型作業のワークフロー |
| **State Manager** | 設定の継続的な維持 | ドリフト防止 |

> 🎯 **試験のポイント**:
>
> - 「SSHキーなしでEC2に接続」「踏み台廃止」→ **Session Manager**
> - 「OSパッチの定期適用」→ **Patch Manager**

**概要**: コスト最適化、パフォーマンス、セキュリティ、耐障害性、サービス制限の5分野でベストプラクティスをチェック。

| サポートプラン | 利用可能なチェック |
|---------------|------------------|
| Basic / Developer | 一部のみ |
| Business以上 | 全項目 |

> 🎯 **試験のポイント**: 「セキュリティグループのポート開放検出」「使用されていないEIP」→ **Trusted Advisor**
